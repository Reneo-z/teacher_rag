# AGENTS.md

> Agent entrypoint for this development project.

## Reading Order

1. `AGENTS.md`
2. `project.md`
3. `dev.md`
4. `env.md`
5. Recent files under `devlogs/`

## Context Rules

- Keep root navigation files concise.
- Put detailed process records in `devlogs/`.
- Put environment facts in `env.md`.
- Put important document pointers in `context/docs.md`.
- Append history; do not overwrite useful context.
- Mark unverified conclusions as unverified.

## Safety

- Do not store secrets in context files.
- Do not modify customer/source materials unless explicitly asked.
- Follow the repository's existing collaboration rules if they conflict with this scaffold.
