# 创建巡检任务

## 概览

本 Skill 引导用户在巡检管理系统中完成“新建巡检任务并指派执行人”的标准流程，覆盖一次性任务与周期性任务两种类型。

## 适用场景

- 需要为某台设备安排一次性巡检任务。
- 需要为某个区域设置周期性巡检任务。
- 需要将巡检任务指派给一个或多个执行人。

## 前置条件

- [ ] 已进入巡检管理系统。
- [ ] 当前账号角色为巡检员或更高。
- [ ] 目标设备已在设备管理中注册。

## 操作步骤

### 步骤 1：进入新建任务页面
- **操作**：在顶部主导航点击“巡检管理”，在左侧子菜单点击“任务管理”，再点击右上角“+ 新建任务”按钮。
- **界面位置描述**：巡检管理 > 任务管理页::右上角“+ 新建任务”按钮
- **截图**（可选）：`assets/step-1.png`

### 步骤 2：填写任务基本信息
- **操作**：在弹出的表单中填写任务名称、任务类型和优先级。
- **界面位置描述**：新建任务弹窗::基本信息区域
- **说明**：任务名称建议使用“设备名 + 巡检周期 + 日期”的格式，便于后续检索。

### 步骤 3：选择巡检对象
- **操作**：点击“选择设备”按钮，在设备树中勾选目标设备，点击“确定”。
- **界面位置描述**：新建任务弹窗::设备选择区

### 步骤 4：指派执行人
- **操作**：在“执行人”下拉框中选择巡检员；如需多人协同，点击“+”继续添加。
- **界面位置描述**：新建任务弹窗::执行人选择区

### 步骤 5：配置周期与提醒
- **操作**：如任务类型选择“周期性”，配置周期和提前提醒时间；一次性任务可跳过本步。
- **界面位置描述**：新建任务弹窗::周期与提醒区

### 步骤 6：提交并发布
- **操作**：点击表单底部“保存并发布”按钮。
- **界面位置描述**：新建任务弹窗底部::“保存并发布”按钮

## 验证方式

1. 页面出现“任务创建成功”提示。
2. 在任务管理列表中可以看到刚创建的任务，状态为“待执行”。
3. 被指派执行人收到系统通知。

## 常见异常 / 错误处理

| 现象 | 原因 | 解决办法 |
|---|---|---|
| “+ 新建任务”按钮不可点击 | 当前账号角色权限不足 | 联系管理员调整账号角色 |
| 设备树为空 | 当前账号无设备查看范围 | 联系管理员配置设备访问范围 |
| 提交时报错“执行人不在排班内” | 执行人未排班至该时段 | 先在人员排班中安排班次，或更换执行人 |
| 提示设备已有未完成任务 | 同设备同类型任务未完成 | 先处理已有任务，或调整本次任务对象 |
